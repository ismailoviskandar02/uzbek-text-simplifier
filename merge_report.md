# Отчёт: объединение корпуса `uz_corpus`

## Входные файлы

| Файл | Строк | Есть `domain`? |
|---|---|---|
| `dataset.csv` | 2033 | да |
| `uz_corpus_20260723_1106.csv` | 214 | нет |
| `uz_corpus_20260723_1111.csv` | 395 | нет |
| `uz_corpus_20260723_1117.csv` | 500 | нет |
| **Всего до дедупа** | **3142** | |

Три файла `uz_corpus_*` — последовательные прогоны одного и того же
парсера (`uz_news_parser3.py`) утром 23.07 (kun.uz, uza.uz, gazeta.uz,
Wikipedia по 14 категориям). Они сильно пересекаются друг с другом и с
`dataset.csv` (61–106 общих URL между парами файлов).

## Как проходило объединение

**1. Конкатенация** — все 4 файла собраны в один датафрейм.

**2. Дедуп по тексту чанка** (не по `url`+`chunk_id`, т.к. один и тот же
текст в разных прогонах мог получить разный `chunk_id` из-за нарезки на
предложения). При совпадении текста в нескольких файлах оставлена версия
из `dataset.csv` — там уже был проставлен `domain`.
- Удалено дублей: **907**
- Итоговый размер: **2235** строк

**3. Восстановление `domain`** — в трёх файлах `uz_corpus_*` колонки не
было; готовой разметки в парсерах для неё нет (только список
Wikipedia-категорий и RSS-источники без предметных меток). `domain`
восстановлен эвристически по ключевым словам в `title`+`text` (кириллица
и латиница), с fallback на `other`.
- Доклассифицировано так: **202 строки** (помечены `domain_filled_auto =
  True` — можно отфильтровать и перепроверить вручную)
- Остальные 2033 строки сохранили исходный `domain` из `dataset.csv`

> Это **приближённая** классификация на правилах, не идентичная разметке
> `dataset.csv` (там тоже встречаются спорные случаи — например, статья
> о стратегии «Ўзбекистон – 2030» размечена как `technology`).
> Рекомендуется выборочно проверить строки с `domain_filled_auto = True`,
> особенно в доменах `other` / `technology` / `politics`, где выше
> пересечение лексики между доменами.

## На заметку

- Корпус сильно перекошен в сторону `law` (~59%, почти целиком за счёт
  lex.uz) — для fine-tuning mT5-small стоит либо даунсэмплить law, либо
  догонять домены health/technology/education (сейчас <100 строк каждый).
- В файле есть две служебные колонки: `domain_filled_auto` (bool) и
  `origin_file` — можно удалить перед подачей в пайплайн обучения, если
  не нужны.
- Дедуп по тексту не ловит почти-дубликаты (например, слегка обновлённую
  версию той же новости) — при необходимости можно дополнительно
  прогнать fuzzy-dedup по similarity.
- **`domain_report.txt`** (старый файл, разбивка по доменам в разрезе
  source) относится к датасету **до** этого слияния (2033 строки) и
  больше не соответствует финальному `uz_corpus_merged_final.csv`.
  Детальную разбивку по источникам для смёрженного корпуса нужно
  пересчитать заново — здесь приведены только агрегированные итоги.
