# Отчёт: очистка текста от мусора разметки

Относится к датасету на момент 2033 строк (до объединения с
дополнительными прогонами парсера, см. `merge_report.md`).

- Всего строк: **2033**
- Строк, где текст был изменён очисткой: **45**
- Строк, ставших пустыми после очистки: **0**

Удалялись остатки LaTeX/MathML-дампов формул, HTML-теги, обрывки
твиттер-виджетов, цитатные ссылки вида `[10]`, zero-width/BOM-символы.

## Примеры (до → после)

> Примечание: во многих примерах ниже видимый текст «до» и «после»
> совпадает — это артефакт обрезки строки до 180 символов для вывода в
> отчёт. Реальное изменение (обычно невидимый zero-width символ или
> непарная кавычка после арабской вставки) находится дальше в тексте
> чанка и не попадает в показанный фрагмент.

| # | Пример |
|---|---|
| 1 | `Abad (arabcha: ابد‎‎ — ...)` → `Abad (arabcha: ابد — ...)` — убран лишний невидимый маркер направления текста после арабской вставки |
| 2 | `Abdullatif al-Bag'dodiy (arabcha: عبد اللطيف البغدادي‎‎ — ...)` → аналогичная чистка |
| 3 | `Abu-Dabi universiteti (ADU) (arabcha: جامعة أبوظبي‎‎)` → `(arabcha: جامعة أبوظبي)` |
| 4 | `...eng yaxshi muassasalardan biri bo'lgan muassasadir".[10] = Ak...` → цитатная ссылка `[10]` удалена |
| 5 | Ещё ~40 подобных случаев — преимущественно арабские вставки в статьях Wikipedia с лишними невидимыми Unicode-маркерами |

Полный список всех 45 изменённых строк — в исходном `cleaning_report.txt`
(включён при необходимости построчной сверки).

## На заметку

- Все найденные искажения — технический мусор (невидимые символы,
  ссылки, HTML), смысл текста не менялся.
- 0 строк опустело — очистка не привела к потере контента.
- Эта чистка предшествовала объединению корпусов; после слияния с
  дополнительными прогонами парсера (`merge_report.md`) отдельная
  проверка на те же паттерны не проводилась — стоит прогнать при
  генерации финального датасета для обучения.
