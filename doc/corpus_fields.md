# Формулы полей корпуса `uz_corpus_*.csv`

## Обозначения

- `T` — текст чанка (поле `text`)
- `W = {w₁, w₂, ..., wₙ}` — список слов чанка, полученный токенизацией (способ токенизации указан отдельно для каждой группы полей ниже, т.к. он отличается)
- `S = {s₁, s₂, ..., sₘ}` — список предложений чанка
- `n = |W|` — число слов
- `m = |S|` — число предложений
- `F(w)` — частота слова `w` во всём корпусе (частотный словарь, построенный по всем чанкам)

---

## 1. `word_count`, `length`

**Токенизация:** `T.split()` — разбиение по пробельным символам (без regex, без фильтрации пунктуации).

```
word_count = |T.split()|

length = |T|          (число символов в тексте чанка)
```

---

## 2. `avg_sentence_len`, `avg_word_len`, `rare_word_ratio`, `complexity_score`

**Токенизация слов** (регулярное выражение, только буквы, без цифр/пунктуации):

```
WORD_RE = [a-zA-Zʻʼ'\u0400-\u04FF]+
W = { w.lower() : w ∈ findall(WORD_RE, T) }
```

**Токенизация предложений:**

```
S = split(T, [.!?]+)   (непустые фрагменты после разбиения по .!?)
```

### avg_sentence_len — средняя длина предложения (в словах)

```
avg_sentence_len = n / m
```

### avg_word_len — средняя длина слова (в символах)

```
avg_word_len = ( Σ |wᵢ| for wᵢ ∈ W ) / n
```

### rare_word_ratio — доля редких слов

Слово считается «редким», если оно встречается во всём корпусе не более `rare_threshold` раз (по умолчанию `rare_threshold = 3`).

```
rare_count = |{ w ∈ W : F(w) ≤ 3 }|

rare_word_ratio = rare_count / n
```

### complexity_score — итоговая сложность

Три фактора нормируются в диапазон [0, 1] по эмпирически подобранным потолкам, затем суммируются с весами:

```
norm_sentence_len = min(avg_sentence_len / 25, 1)   # потолок: 25 слов/предложение
norm_word_len     = min(avg_word_len / 10, 1)       # потолок: 10 символов/слово
norm_rare_ratio   = min(rare_word_ratio, 1)          # уже в [0,1] по построению

complexity_score = 0.4 · norm_sentence_len
                  + 0.3 · norm_rare_ratio
                  + 0.3 · norm_word_len
```

Диапазон: `0` (просто) — `1` (сложно). Веса (0.4 / 0.3 / 0.3) и потолки (25, 10) — эвристика, подобранная вручную под этот корпус, не общепринятая метрика читаемости.

> **Важно про рассинхрон в текущем файле:** после чистки текста (`clean_corpus.py`) поля `word_count`, `length`, `avg_sentence_len`, `avg_word_len` были пересчитаны **упрощённым способом** (см. раздел 3 ниже), а `rare_word_ratio` и `complexity_score` остались из исходного прогона парсера — на нечищенном тексте и со старым частотным словарём `F(w)`. Формулы разные, поэтому для строгой методологической чистоты все шесть полей нужно пересчитать заново одним способом (раздел 2, с перестройкой `F(w)` по очищенному корпусу).

## 3. Формулы, которыми `word_count`/`length`/`avg_*` были пересчитаны при чистке (`clean_corpus.py`)

Отличаются от раздела 2 токенизацией — используется наивный `.split()` вместо regex, а границы предложений учитывают многоточие `…`:

```
W' = T.split()
n' = |W'|

S' = split(T, регулярка на позициях после [.!?…] + пробел)
m' = |S'|

word_count       = n'
length           = |T|
avg_sentence_len = n' / m'
avg_word_len     = ( Σ |wᵢ'| for wᵢ' ∈ W' ) / n'
```

## 4. `domain` (добавлено отдельно, не формула — keyword-скоринг)

```
score(domain) = 3 · [заголовок содержит ключевое слово домена]
              + 1 · [текст чанка содержит ключевое слово домена]
              (суммируется по всем ключевым словам домена)
```

Скоринг считается **на уровне статьи** — все чанки одного `url` суммируют счёт вместе, итоговый `domain` присваивается сразу всем чанкам этой статьи.

```
domain(article) = argmax_d score(d)
```

При равенстве максимума — приоритет:
`law > health > science > education > economy > politics > sport > technology > culture`.

Если счёт по всем доменам равен 0 → `domain = "other"`.
Исключение: `source = "lex.uz"` → `domain = "law"` всегда, без скоринга (это сайт с текстами законов).

Список ключевых слов по доменам — в `add_domain.py` (`DOMAIN_KEYWORDS`).
