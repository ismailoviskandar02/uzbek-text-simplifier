# Uzbek Text Simplifier — сводка проекта

Капстоун-проект: сервис, упрощающий сложные узбекские тексты (юридические
документы, официальные новости, гос. инструкции) для обычного читателя,
с опциональным переводом результата на русский/английский.

## Идея

Выбор пал на **NLP для узбекского языка** — нишу почти без готовых
инструментов (в отличие от английского). Из вариантов (классификация
новостей / тональность / NER / упрощение текста) выбрано **text
simplification** — сильнее по бизнес-ценности и без готовых конкурентов
под узбекский.

**Архитектура:**
1. Сбор parallel-корпуса (сложный ↔ простой текст)
2. Fine-tuning mT5-small на корпусе
3. NLLB-200 — опциональный перевод упрощённого текста на ru/en
4. Демо на FastAPI + Streamlit/Gradio, деплой на HuggingFace Spaces

## Сбор корпуса

Три парсера + постобработка, единая CSV-схема (source, title, url,
chunk_id, text, word_count, length, avg_sentence_len, avg_word_len,
rare_word_ratio, complexity_score, domain). Чанкинг — 100–200 слов по
границам предложений.

**Источники:**
- Новости: kun.uz, gazeta.uz, uza.uz (RSS)
- Wikipedia: 14 категорий, рекурсивный обход подкатегорий (до 2 уровней)
- lex.uz: юридический корпус, 9 документов (ручной seed-список) — самый
  "сложный" стилистический полюс

Сложность текста считается тремя эвристиками (без готовых NLP-инструментов
под узбекский): средняя длина предложения, доля редких слов, средняя длина
слова → взвешенный `complexity_score`.

После склейки корпусов проведена постобработка: вычищены LaTeX/HTML-мусор,
твиттер-остатки, цитатные ссылки; добавлена колонка `domain`
(keyword-классификация по 9 категориям + other, на уровне статьи).

**Итог:** 2033 чанка из 230 статей (wikipedia 690, uza.uz 65, gazeta.uz 76,
kun.uz 68, lex.uz 1134). `complexity_score`: медиана 0.502. Домены: law
63.1%, остальное — наука/экономика/культура/спорт и др. Датасет чист: 0
пропусков, 0 повреждённых строк, 0 вики-артефактов.

Полные формулы полей — в `corpus_fields.md`.

## Проверка качества

Прогнан через DataStory (Tkinter + Gemini) и ручной аудит. Автоматические
"тревоги" AI-анализа (уникальность `text`, "выбросы" в длине чанков,
будущие даты 2026) — ложные срабатывания, ожидаемые для NLP-датасета.
Ручной аудит дополнительно выявил утечку разметки (LaTeX/HTML/твиттер),
не пойманную парсером и DataStory — исправлено (см. `cleaning_report.txt`).
